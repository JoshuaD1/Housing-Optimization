# Subsidized Housing Coverage Optimization Model

A county-level integer linear programming (ILP) model that maximizes
the number of low-income renter households served subject to a per-county
public subsidy budget.  The model allocates four housing types across four
income tiers and scales construction costs by a state RS Means index.

Companion to:

> *What Incentives Sustain Persistent Participation in Demand Response
> under Real-World Household, Contractual, and Market Constraints?*
> — Joshua Darville, PhD (working paper)

---

## Model Overview

### Decision variables

| Variable | Type | Description |
|---|---|---|
| `x[i][h]` | Integer ≥ 0 | Households of income group *i* placed in housing type *h* |
| `y[i][h]` | Binary | Indicator: `x[i][h] > 0` |
| `s[h]` | Integer ≥ 0 | Total units of type *h* built |
| `subcost[i][h]` | Continuous ≥ 0 | Subsidy cost for group *i* in type *h* |

### Objective

Maximise total households served (weighted by unit type):

$$\max \sum_i \sum_h \omega_h \cdot x_{ih}$$

### Key constraints

| Eq. | Constraint | Meaning |
|---|---|---|
| 2 | Income-mix | Weighted income share ≤ α · total served |
| 3 | Demand cap | Households placed ≤ unmet deficit per tier |
| 4 | Unit count | Units built = sum of placements |
| 7 | Subsidy lower bound | Subsidy ≥ (Op + PMT + ρ·Equity − θ·HAMFI) · x |
| 8 | Indicator linkage | x ≤ D · y |
| 12 | Budget | Total subsidy ≤ B per county |

### Income tiers

| Tier | HAMFI threshold | Default income |
|---|---|---|
| ELI (Extremely Low) | ≤ 30% | $20,000 |
| VLI (Very Low) | ≤ 50% | $35,000 |
| LI (Low) | ≤ 80% | $50,000 |
| NLI (Non-Low) | ≤ 100% | $70,000 |

### Housing types

| Type | Unit | Cap cost | Annual rent |
|---|---|---|---|
| 1 | Room | $180,000 | $10,500 |
| 2 | Studio | $220,000 | $12,000 |
| 3 | 1-Bedroom | $280,000 | $14,400 |
| 4 | 2-Bedroom | $350,000 | $17,400 |

## Quick Start

### 1. Clone

```bash
git clone https://github.com/<your-username>/housing-coverage-model.git
cd housing-coverage-model
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Download data

See [`data/README.md`](data/README.md) for download links and placement.

### 4. Run the optimization model

```bash
python src/housing_model.py
```

Optional flags:

```bash
python src/housing_model.py \
  --input  data/County_CHAS_with_FMR_RSMeans.csv \
  --output outputs/coverage_results.csv \
  --budget 5000000
```

### 5. Generate maps

```python
from src.map_functions import load_and_merge_data, generate_all_maps_at_level

gdf = load_and_merge_data(
    results_csv   = "outputs/coverage_results.csv",
    geocodes_xlsx = "data/all-geocodes-v2020.xlsx",
    county_shp    = "data/cb_2024_us_county_5m/cb_2024_us_county_5m.shp",
    output_dir    = "outputs/maps",
    scenario_tag  = "Base",
)

for level in ["county", "state"]:
    generate_all_maps_at_level(gdf, level=level)
```

## Outputs

The model writes the following files to `outputs/`:

| File | Description |
|---|---|
| `coverage_results.csv` | Per-county ILP results (status, households served, subsidy paid, unit mix) |
| `Base_coverage_by_county.csv` | Results merged with geographic identifiers |
| `county_coverage_rate_ELI.png` | County-level ELI coverage choropleth |
| `state_coverage_rate_ELI.png` | State-level ELI coverage choropleth |
| `county_subsidy_efficiency_map.png` | Households served per $1M (county) |
| `state_subsidy_efficiency_map.png` | Households served per $1M (state) |
| `county_unit_type_mix_stacked.png` | Unit-type stacked bar chart |
| `*_coverage_summary.csv` | Aggregate coverage stats by level |

## Configuration

All model parameters live at the top of `src/housing_model.py`:

| Parameter | Default | Description |
|---|---|---|
| `BUDGET_PER_COUNTY` | $5,000,000 | Subsidy budget per county |
| `RENT_BURDEN_PCT` | 0.30 | Max rent as share of income (θ) |
| `AVG_INCOME_PCT` | 0.60 | Income-mix constraint (α) |
| `INCOME_PARAMS[i]['rho']` | 0.05 | Required equity return per tier |
